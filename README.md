
# 🛡️ Day 03 – Human, Process & Technology in SOC

## 🚀 What I Learned Today

Today I learned about one of the most important concepts in a Security Operations Center (SOC):

# Human + Process + Technology

A SOC is not only about security tools.  
It works properly when skilled people, structured workflows, and security technologies work together.

---

# 👨 Human

The “Human” part represents the people working inside the SOC team.

Different roles have different responsibilities:

- **L1 Analysts** → Monitor alerts and identify suspicious activity
- **L2 Analysts** → Perform deeper investigations
- **Security Engineers** → Manage and improve security solutions
- **SOC Managers** → Handle team operations and coordination

Every role contributes to protecting the organization.

---

# ⚙️ Process

Process means following proper workflows and procedures during investigations.

One important process I learned today is:

## Alert Triage

When an alert appears, analysts try to understand:

- What happened?
- When did it happen?
- Where did it happen?
- Who was involved?
- Why did it happen?

This helps analysts prioritize and investigate alerts correctly instead of ignoring them.

I also learned that processes like:
- Reporting
- Incident Response
- Documentation

are very important in a SOC environment.

---

# 💻 Technology

Technology represents the security tools used inside a SOC.

Some common technologies include:

- **SIEM** → Security monitoring and log analysis
- **EDR** → Endpoint monitoring and threat detection
- **Firewall** → Filters and monitors network traffic
- **IDS/IPS** → Detects and prevents suspicious activity

These tools help SOC teams detect, monitor, and respond to threats more effectively.

---

# 📌 Key Takeaway

A strong SOC is built on three things working together:

✔ Skilled People  
✔ Proper Processes  
✔ Security Technologies

Without balance between these three pillars, security operations become difficult.
